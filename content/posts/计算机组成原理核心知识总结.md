## 一、计算机系统概述

### 1.1 冯·诺依曼体系结构

计算机的经典模型，几乎所有现代计算机都基于此思想：

- **五大部件**：运算器、控制器、存储器、输入设备、输出设备
- **存储程序**：指令和数据以同等地位存放在存储器中，按地址访问
- **指令驱动**：程序由指令组成，指令按顺序执行（除非跳转）

**核心特征**：指令和数据都用二进制表示，通过地址总线/数据总线/控制总线连接各部件。

### 1.2 计算机硬件架构

```
CPU（运算器 + 控制器）
  ├── 运算器（ALU）：算术逻辑运算
  │   ├── ACC（累加器）
  │   ├── MQ（乘商寄存器）
  │   └── X（操作数寄存器）
  ├── 控制器：取指、译码、执行
  │   ├── PC（程序计数器）
  │   ├── IR（指令寄存器）
  │   └── CU（控制单元）
  └── 寄存器组
存储器（内存）— 指令 + 数据
I/O 设备 — 与外设交互
```

### 1.3 计算机性能指标

| 指标 | 定义 | 单位 |
|------|------|------|
| **主频** | CPU 时钟频率 | Hz |
| **CPI** | 每条指令所需时钟周期数 | cycles/instruction |
| **MIPS** | 每秒百万条指令 | 10⁶ IPS |
| **MFLOPS** | 每秒百万次浮点运算 | 10⁶ FLOPS |
| **响应时间** | 完成一个任务的总时间 | s |
| **吞吐率** | 单位时间内完成的任务数 | tasks/s |

**核心公式**：
- 程序执行时间 = 指令数 × CPI × 时钟周期 = 指令数 × CPI / 主频

## 二、数据的表示与运算

### 2.1 数制与编码

- **二进制（Binary）**：计算机唯一能直接识别的进制
- **八进制（Octal）**：3 位二进制对应 1 位八进制
- **十六进制（Hexadecimal）**：4 位二进制对应 1 位十六进制
- **BCD 码**：用 4 位二进制表示 1 位十进制数（如 8421 码）
- **ASCII 码**：7 位编码，共 128 个字符

### 2.2 机器数的表示

#### 原码、反码、补码、移码

| 码制 | 正数表示 | 负数表示 | 特点 |
|------|----------|----------|------|
| **原码** | 符号位为 0 | 符号位为 1，数值位不变 | 零有两种表示，加减复杂 |
| **反码** | 同原码 | 符号位为 1，数值位取反 | 零有两种表示 |
| **补码** | 同原码 | 反码末位加 1 | **主流方案**，零唯一，加减统一 |
| **移码** | 补码符号位取反 | 补码符号位取反 | 便于比较大小（浮点阶码常用） |

**关键结论**：
- 补码的补码 = 原码
- `[X]补 + [Y]补 = [X+Y]补`（加减法统一）
- 8 位补码范围：-128 ~ +127
- 16 位补码范围：-32768 ~ +32767

### 2.3 定点数与浮点数

#### 定点数

- **定点整数**：小数点固定在最低位之后
- **定点小数**：小数点固定在符号位之后
- **优点**：运算简单，硬件实现容易
- **缺点**：表示范围小，精度有限

#### 浮点数（IEEE 754 标准）

**32 位单精度格式**：

```
[1位符号位S]  [8位阶码E(移码)]  [23位尾数M(原码)]
```

**64 位双精度格式**：

```
[1位符号位S]  [11位阶码E(移码)]  [52位尾数M(原码)]
```

**真值公式**：
- 规格化数：`(-1)^S × 1.M × 2^(E - bias)`
- 偏置值 bias：单精度 127，双精度 1023

**特殊值**：
| E | M | 表示 |
|---|----|------|
| 全 0 | 全 0 | ±0 |
| 全 0 | 非 0 | 非规格化数 |
| 全 1 | 全 0 | ±∞ |
| 全 1 | 非 0 | NaN |

**浮点数加减步骤**：对阶 → 尾数求和 → 规格化 → 舍入 → 判溢出

### 2.4 算术逻辑单元 ALU

- **核心组成**：加法器（全加器 FA、串行进位/并行进位）、逻辑运算电路
- **并行进位（先行进位 CLA）**：通过进位生成函数 `Gi = Ai·Bi` 和进位传递函数 `Pi = Ai⊕Bi` 提前计算进位，减少延迟
- **ALU 芯片**：如 74181（4 位 ALU），可级联扩展

### 2.5 数据校验

- **奇偶校验**：增加一位使 1 的个数为奇数/偶数，只能检错不能纠错
- **海明码**：分组奇偶校验，可检错 1 位、纠错 1 位
  - 校验位位数 k 满足：`2^k ≥ n + k + 1`
- **CRC 循环冗余校验**：基于模 2 运算，检错能力强，广泛用于网络和存储

## 三、存储器层次结构

### 3.1 存储器层次结构全景

```
        ┌──────────┐
        │  寄存器   │  最快（~1ns），最小（~KB），CPU 内部
        ├──────────┤
        │   Cache   │  SRAM，~几ns，~MB，芯片内/主板
        ├──────────┤
        │   主存    │  DRAM，~几十ns，~GB
        ├──────────┤
        │  虚存磁盘  │  机械/固态，ms 级，TB 级
        └──────────┘
```

**局部性原理**（整个体系的基础）：
- **时间局部性**：刚被访问过的数据很可能再次被访问
- **空间局部性**：刚被访问的数据附近的数据很可能被访问

### 3.2 主存储器

#### SRAM vs DRAM

| 特性 | SRAM（静态 RAM） | DRAM（动态 RAM） |
|------|------------------|-------------------|
| 存储单元 | 6 个 MOS 管（触发器） | 1 个 MOS 管 + 1 个电容 |
| 速度 | 快 | 慢 |
| 集成度 | 低 | 高 |
| 功耗 | 大 | 小 |
| 刷新 | 不需要 | 需要（~2ms） |
| 用途 | Cache | 主存 |
| 成本 | 高 | 低 |

#### DRAM 刷新方式

- **集中刷新**：定期间隔刷新全部行，存在"死区"
- **分散刷新**：将刷新分散到每个存取周期后，无死区但慢
- **异步刷新**：二者折中，最常用

#### ROM 类型

- **MROM**（掩膜 ROM）：出厂固化，不可改
- **PROM**：一次可编程
- **EPROM**：可擦除（紫外线），可重编程
- **EEPROM**：电可擦除
- **Flash Memory**：闪存（U 盘、SSD），快速电擦除

### 3.3 主存与 CPU 的连接

- **地址线**：决定寻址范围（n 条线 → 2ⁿ 个地址）
- **数据线**：决定一次读写的数据位数
- **控制线**：读/写信号、片选信号（CS/CE）
- **字扩展**：增加存储字数（通过片选线）
- **位扩展**：增加存储字长（多片并联数据线）
- **字位同时扩展**：先位扩再字扩

### 3.4 Cache 高速缓存

#### 基本原理

- **命中率 h**：CPU 访问 Cache 命中的概率
- **平均访问时间** = h × Tc + (1-h) × Tm
- **访问效率** = Tc / 平均访问时间

#### 地址映射方式

| 映射方式 | 原理 | 冲突率 | 利用率 | 复杂度 |
|----------|------|--------|--------|--------|
| **直接映射** | 主存块 → Cache 唯一位置 | 高 | 低 | 简单 |
| **全相联映射** | 主存块 → Cache 任意位置 | 低 | 高 | 复杂 |
| **组相联映射** | 主存块 → 特定组内任意位置 | 中 | 中 | 适中（常用） |

#### 替换算法

- **LRU**（最近最少使用）：替换最久未被访问的块，命中率最高
- **FIFO**：替换最早进入的块
- **随机替换**：简单但命中率不稳定

#### 写策略

- **写直达（Write Through）**：同时写 Cache 和主存，简单但慢
- **写回（Write Back）**：只写 Cache，被替换时写回主存 — 快但需脏位标记
- **写分配/非写分配**：写未命中时是否将块调入 Cache

### 3.5 虚拟存储器

- **目的**：逻辑上扩充主存容量，使程序可在比物理内存小的空间运行
- **页式虚拟存储**：主存和辅存分为固定大小的页（Page），通过页表映射
- **页表**：存储虚页号 → 实页号的映射关系，包含有效位、脏位、访问权限等
- **TLB（快表）**：页表的 Cache，加速地址转换
- **段式/段页式**：按逻辑段划分，结合页式管理

## 四、指令系统

### 4.1 指令格式

**基本结构**：`[操作码 OP] [地址码 A]`

按地址码数量分类：

| 类型 | 格式 | 特点 |
|------|------|------|
| **三地址** | `OP A1, A2, A3` → A1 OP A2 → A3 | 最直观 |
| **二地址** | `OP A1, A2` → (A1) OP (A2) → A1 | 常用（x86） |
| **一地址** | `OP A` → ACC OP (A) → ACC | 隐含累加器 |
| **零地址** | `OP` | 操作数隐含在栈顶（用于栈结构） |

### 4.2 寻址方式

| 寻址方式 | 操作数地址计算 | 用途 |
|----------|---------------|------|
| **立即寻址** | 操作数 = A（地址码就是操作数本身） | 常量 |
| **直接寻址** | EA = A | 固定地址 |
| **间接寻址** | EA = (A)（A 指向的存储单元存放有效地址） | 指针 |
| **寄存器寻址** | 操作数在寄存器 Ri 中 | 快速存取 |
| **寄存器间接寻址** | EA = (Ri) | 灵活寻址 |
| **变址寻址** | EA = (IX) + A | 数组访问 |
| **基址寻址** | EA = (BR) + A | 程序重定位 |
| **相对寻址** | EA = (PC) + A（偏移量） | 转移指令 |
| **堆栈寻址** | 栈顶（隐含） | 函数调用 |

### 4.3 指令类型

- **数据传送类**：MOV, LOAD, STORE, PUSH, POP
- **算术运算类**：ADD, SUB, MUL, DIV, INC, DEC
- **逻辑运算类**：AND, OR, NOT, XOR, SHL, SHR
- **程序控制类**：JMP, CALL, RET, INT（中断返回）
- **输入输出类**：IN, OUT
- **处理器控制类**：HLT, NOP, CLC, STC

### 4.4 CISC vs RISC

| 特性 | CISC（复杂指令集） | RISC（精简指令集） |
|------|-------------------|---------------------|
| 指令数量 | 多（>200） | 少（~50-100） |
| 指令长度 | 不定长 | 定长（多为 32 位） |
| 寻址方式 | 多种 | 少（load/store） |
| 执行方式 | 微程序控制 | 硬布线控制为主 |
| 流水线 | 难以实现 | 易于实现 |
| 典型代表 | x86/x64 | ARM, RISC-V, MIPS |

## 五、中央处理器 CPU

### 5.1 CPU 的功能与组成

**功能**：
- **取指令**：从主存中取出指令（PC → MAR → 读主存 → MDR → IR）
- **分析指令**：译码（IR 中操作码部分送 CU）
- **执行指令**：按操作码执行相应微操作
- **中断处理**：响应和处理各类中断

**组成**：
- **ALU**：运算器，执行算术和逻辑运算
- **CU（控制单元）**：产生控制信号，协调各部件
- **寄存器**：PC, IR, MAR, MDR, ACC, PSW（程序状态字）...

### 5.2 指令周期

一条指令从取指到执行完成的完整过程：

```
取指周期 → 间址周期（如有间接寻址） → 执行周期 → 中断周期（如有中断）
```

**四级时序系统**：
- **机器周期（CPU 周期）**：完成一次基本操作（如访存）
- **时钟周期（节拍 T）**：CPU 的最小时间单位
- **指令周期**：完成一条指令的时间
- **工作脉冲 P**：比时钟周期更窄的脉冲

**指令周期 = N 个机器周期 = N×M 个时钟周期**

### 5.3 微程序控制器

- **思想**：每条机器指令由一段微程序实现，每条微指令包含若干个微命令
- **微命令**：直接作用于硬件的最小控制信号
- **微操作**：微命令执行的结果（如打开某门、传送数据）
- **微指令格式**：
  - **水平型**：每位对应一个微命令，并行度高，但字长大
  - **垂直型**：编码表示微命令，字长短，但需要译码
- **控制存储器（CM）**：存放所有微程序，通常用 ROM
- **微地址形成**：操作码映射 → 微程序入口地址 → 顺序执行（+1）或转移

### 5.4 硬布线控制器

- 直接用逻辑门电路产生控制信号
- **优点**：速度快（微程序需要访问 CM）
- **缺点**：设计复杂、修改困难
- 现代高性能 CPU 多采用硬布线（RISC 体系）

### 5.5 指令流水线

#### 经典五级流水线

```
取指 IF → 译码 ID → 执行 EX → 访存 MEM → 写回 WB
```

**理想加速比**：`k 级流水线，n 条指令 → 加速比 ≈ k（当 n 很大时）`

#### 流水线冲突（Hazards）

| 冲突类型 | 描述 | 解决方法 |
|----------|------|----------|
| **结构冲突** | 硬件资源竞争（如单端口存储器） | 增加资源、流水化功能部件 |
| **数据冲突** | 后一条指令依赖前一条的结果 | 旁路转发（Forwarding）、插入气泡（Bubble） |
| **控制冲突** | 转移指令导致取指地址不确定 | 分支预测、延迟转移 |

**数据冲突的三种情况**：
- **RAW**（读后写，真相关）：后继指令需要前驱指令的结果 — 必须等待
- **WAR**（写后读）：后继指令写入前驱指令要读的寄存器 — 流水线中可实现
- **WAW**（写后写）：两条指令写同一寄存器 — 需保证顺序

#### 流水线性能指标

- **吞吐率 TP**：单位时间完成的指令数
- **加速比 S**：非流水时间 / 流水时间
- **效率 E**：流水线各段的利用率

## 六、总线系统

### 6.1 总线概述

**总线**是连接计算机各部件的公共信息通道，分时共享。

**分类**：
- **片内总线**：CPU 芯片内部连接
- **系统总线**：连接 CPU、主存、I/O 接口
  - **数据总线（DB）**：双向，传输数据，宽度通常 = 机器字长
  - **地址总线（AB）**：单向（CPU→其他），决定寻址范围
  - **控制总线（CB）**：传输控制信号（读/写、中断请求等）
- **通信总线**：计算机之间通信

### 6.2 总线特性与性能

| 特性 | 说明 |
|------|------|
| **宽度** | 数据线的根数（如 32 位、64 位） |
| **频率** | 总线时钟频率 |
| **带宽** | 最大传输速率 = 宽度 × 频率（B/s） |
| **复用** | 地址线和数据线分时共用（减少引脚） |

### 6.3 总线仲裁

- **集中式仲裁**：
  - **链式查询**：简单，但对故障敏感，优先级固定
  - **计数器定时查询**：灵活但需更多控制线
  - **独立请求**：响应快，优先级可编程，线多
- **分布式仲裁**：各设备自己决定

### 6.4 总线标准

| 标准 | 类型 | 带宽 |
|------|------|------|
| **PCI** | 并行系统总线 | 133 MB/s（32bit/33MHz） |
| **PCIe** | 高速串行总线 | 单通道 ~250MB/s（v1.0）→ GB/s 级 |
| **USB** | 外部串行总线 | 480Mbps（2.0）→ 10Gbps+（3.x） |
| **SATA** | 存储接口 | 1.5Gbps ~ 6Gbps |

## 七、输入输出系统

### 7.1 I/O 接口

**功能**：
- 数据缓冲（速度匹配）
- 信号电平/格式转换
- 设备选择和地址译码
- 中断/DMA 控制逻辑

**I/O 端口编址方式**：
- **独立编址**（I/O 映射）：专用 I/O 指令（IN/OUT），地址空间独立（如 x86）
- **统一编址**（存储器映射）：I/O 端口和主存统一编址，用访存指令访问（如 ARM）

### 7.2 数据传送方式

#### 程序查询方式

- CPU 循环检测外设状态（轮询）
- **优点**：简单
- **缺点**：CPU 利用率低，响应慢

#### 程序中断方式

- 外设准备好后向 CPU 发出中断请求
- CPU 响应后执行中断服务程序
- **中断响应过程**：关中断 → 保护现场 → 判中断源 → 执行服务程序 → 恢复现场 → 开中断 → 返回
- **中断判优**：同时有多个中断请求时决定响应顺序（软件查询 / 硬件排队 / 中断控制器如 8259A）

#### DMA 方式

- 外设与主存之间直接传送数据，无需 CPU 干预
- **DMA 控制器的三个工作阶段**：
  1. **预处理**：CPU 初始化 DMA 控制器（传送方向、内存地址、长度等）
  2. **数据传送**：DMA 控制器接管总线，进行数据传输（周期挪用/交替访问）
  3. **后处理**：传送完毕向 CPU 发出 DMA 结束中断
- **优点**：适合高速、批量数据传送（磁盘 I/O）

#### 三种方式对比

| 方式 | CPU 参与度 | 数据传送率 | 适用场景 |
|------|-----------|-----------|----------|
| 程序查询 | 全程参与 | 低 | 低速外设（键盘） |
| 程序中断 | 传送时参与 | 中 | 中低速、随机数据 |
| DMA | 仅在初始/结束时参与 | 高 | 高速批量传送（磁盘） |

## 八、并行体系结构导论

### 8.1 Flynn 分类法

按指令流和数据流的多重性分类：

| 类型 | 指令流 | 数据流 | 典型实例 |
|------|--------|--------|----------|
| **SISD** | 单 | 单 | 传统单核 CPU |
| **SIMD** | 单 | 多 | 向量处理器、GPU |
| **MISD** | 多 | 单 | 极罕见（脉动阵列近似） |
| **MIMD** | 多 | 多 | 多核 CPU、分布式系统 |

### 8.2 多核与多处理器

- **SMP（对称多处理）**：多个相同处理器共享同一主存，由同一 OS 管理
- **UMA（统一存储访问）**：所有处理器访问主存时间相同
- **NUMA（非统一存储访问）**：每个处理器有本地内存，访问远方内存更慢
- **多核处理器**：多个 CPU 核集成在同一芯片上，共享或独立 Cache

### 8.3 硬件多线程

- **细粒度多线程**：每周期切换线程，隐藏所有停顿
- **粗粒度多线程**：仅在长延时事件（如 Cache 缺失）时切换
- **同时多线程（SMT）**：同一周期可发射多个线程的指令（如 Intel 超线程）

## 九、核心公式与计算汇总

| 公式 | 说明 |
|------|------|
| `执行时间 = IC × CPI × T` | IC=指令数，T=时钟周期 |
| `MIPS = 主频 / (CPI × 10⁶)` | 每秒百万指令数 |
| `片选地址范围` | 由高位地址线经译码器选定芯片 |
| `Cache 容量 = 标记 + 数据 + 有效位 + 脏位` | 需加上状态位 |
| `Cache 命中率 h = N命中 / N总访问` | |
| `平均访问时间 = h×Tc + (1-h)×(Tc+Tm)` | 同时访问 Cache 和主存 |
| `海明码校验位 k：2ᵏ ≥ n+k+1` | n 为数据位数 |
| `流水线加速比 ≈ k（n→∞）` | k 为流水线级数 |
| `总线带宽 = 数据宽度 × 总线频率` | |
| `浮点数真值 = (-1)ˢ × 1.M × 2^(E-bias)` | IEEE 754 |
