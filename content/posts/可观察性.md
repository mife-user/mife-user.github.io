# 可观察性技术栈详解

可观察性是现代分布式系统中不可或缺的一部分，它通过收集、分析和可视化系统的运行数据，帮助我们理解系统行为、排查问题和优化性能。本文将详细介绍可观察性领域的四大核心工具：Prometheus（指标监控）、Loki（日志聚合）、Jaeger（分布式追踪）和Grafana（可视化），并提供完整的Docker Compose部署方案。

## Prometheus - 指标监控系统

### 架构概述

Prometheus是一个开源的监控系统和时间序列数据库，主要用于收集和存储系统指标。其核心架构包括：

1. **Prometheus Server**：负责抓取和存储时间序列数据
2. **Exporters**：负责将各种系统和应用的指标转换为Prometheus可识别的格式
3. **Alertmanager**：处理告警并发送通知
4. **Pushgateway**：用于临时任务的指标收集

Prometheus采用拉取（pull）模式获取指标，通过HTTP协议从目标服务中抓取数据。它使用一种名为PromQL的查询语言来分析和查询指标数据。

### 核心组件

- **Prometheus Server**：核心组件，包含数据采集、存储和查询功能
- **Service Discovery**：自动发现需要监控的目标
- **TSDB**：内置的时间序列数据库，用于存储指标数据
- **Alerting Rules**：定义告警规则

## Loki - 日志聚合系统

### 架构概述

Loki是由Grafana Labs开发的一个水平可扩展、高可用性的日志聚合系统。它的设计理念是"像Prometheus一样处理日志"，具有以下特点：

1. **标签索引**：与Prometheus类似，Loki使用标签来索引日志流，而不是全文索引
2. **高效存储**：通过压缩和去重，减少存储成本
3. **与Grafana集成**：无缝集成到Grafana中，提供统一的可视化体验

### 核心组件

- **Promtail**：日志收集代理，负责将日志发送到Loki
- **Loki**：主服务器，负责存储和索引日志
- **Grafana**：用于查询和可视化日志

Loki的架构设计使其特别适合处理大规模的日志数据，同时保持查询性能和存储效率的平衡。

## Jaeger - 分布式追踪系统

### 架构概述

Jaeger是一个开源的分布式追踪系统，用于监控和排查微服务架构中的性能问题。它由Uber开发并开源，后来成为CNCF的毕业项目。

Jaeger的核心架构包括：

1. **Jaeger Client**：集成到应用程序中的客户端库，用于生成和发送追踪数据
2. **Jaeger Collector**：接收和处理追踪数据
3. **Jaeger Query**：提供UI界面，用于查询和可视化追踪数据
4. **Storage**：存储追踪数据，支持多种后端如Cassandra、Elasticsearch等

### 核心组件

- **Agent**：运行在每个主机上，接收本地客户端发送的追踪数据并批量发送到Collector
- **Collector**：接收追踪数据，进行处理和聚合，然后存储到后端存储
- **Query**：提供API和UI，用于查询和展示追踪数据
- **Ingester**：处理和索引追踪数据（在某些存储后端中使用）

## Grafana - 可视化平台

### 架构概述

Grafana是一个开源的监控和可视化平台，支持多种数据源，包括Prometheus、Loki、Jaeger等。它提供了丰富的可视化选项和强大的查询编辑器。

Grafana的核心架构包括：

1. **Grafana Server**：主服务器，提供Web界面和API
2. **Data Sources**：数据源配置，连接到各种监控系统
3. **Dashboards**：可视化仪表盘，展示监控数据
4. **Alerting**：告警系统，基于监控数据触发告警

### 核心功能

- **统一界面**：为不同的监控系统提供统一的可视化界面
- **灵活的仪表盘**：支持多种图表类型和布局
- **强大的查询编辑器**：针对不同数据源的专用查询编辑器
- **告警管理**：集中管理来自不同系统的告警

## 完整的Docker Compose部署方案

下面提供一个完整的Docker Compose配置，用于部署Prometheus、Loki、Jaeger和Grafana的集成环境：

```yaml
version: '3.8'

services:
  # Prometheus 服务
  prometheus:
    image: prom/prometheus:v2.40.0
    container_name: prometheus
    restart: always
    volumes:
      - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
      - prometheus_data:/prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
      - '--web.console.libraries=/etc/prometheus/console_libraries'
      - '--web.console.templates=/etc/prometheus/consoles'
      - '--web.enable-lifecycle'
    ports:
      - "9090:9090"
    networks:
      - monitoring

  # Grafana 服务
  grafana:
    image: grafana/grafana:9.3.6
    container_name: grafana
    restart: always
    volumes:
      - grafana_data:/var/lib/grafana
      - ./grafana/provisioning:/etc/grafana/provisioning
    environment:
      - GF_SECURITY_ADMIN_USER=admin
      - GF_SECURITY_ADMIN_PASSWORD=admin
      - GF_USERS_ALLOW_SIGN_UP=false
    ports:
      - "3000:3000"
    networks:
      - monitoring

  # Loki 服务
  loki:
    image: grafana/loki:2.7.4
    container_name: loki
    restart: always
    volumes:
      - ./loki/loki.yml:/etc/loki/loki.yml
      - loki_data:/loki
    command:
      - '-config.file=/etc/loki/loki.yml'
    ports:
      - "3100:3100"
    networks:
      - monitoring

  # Promtail 服务（用于收集日志）
  promtail:
    image: grafana/promtail:2.7.4
    container_name: promtail
    restart: always
    volumes:
      - ./promtail/promtail.yml:/etc/promtail/promtail.yml
      - /var/log:/var/log
    command:
      - '-config.file=/etc/promtail/promtail.yml'
    networks:
      - monitoring

  # Jaeger 服务
  jaeger:
    image: jaegertracing/all-in-one:1.39
    container_name: jaeger
    restart: always
    environment:
      - COLLECTOR_ZIPKIN_HOST_PORT=:9411
    ports:
      - "5775:5775/udp"
      - "6831:6831/udp"
      - "6832:6832/udp"
      - "5778:5778"
      - "16686:16686"
      - "14268:14268"
      - "14250:14250"
      - "9411:9411"
    networks:
      - monitoring

  # Node Exporter 服务（用于收集主机指标）
  node_exporter:
    image: prom/node-exporter:v1.3.1
    container_name: node_exporter
    restart: always
    volumes:
      - /proc:/host/proc:ro
      - /sys:/host/sys:ro
      - /:/rootfs:ro
    command:
      - '--path.procfs=/host/proc'
      - '--path.rootfs=/rootfs'
      - '--path.sysfs=/host/sys'
      - '--collector.filesystem.mount-points-exclude=^/(sys|proc|dev|host|etc)($$|/)'
    ports:
      - "9100:9100"
    networks:
      - monitoring

volumes:
  prometheus_data:
  grafana_data:
  loki_data:

networks:
  monitoring:
    driver: bridge
```

### 配置文件

#### Prometheus 配置文件 (`prometheus/prometheus.yml`)

```yaml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

alerting:
  alertmanagers:
    - static_configs:
        - targets:

rule_files:

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  - job_name: 'node_exporter'
    static_configs:
      - targets: ['node_exporter:9100']

  - job_name: 'cadvisor'
    static_configs:
      - targets: ['cadvisor:8080']
```

## 部署步骤

1. **创建目录结构**

```bash
mkdir -p prometheus loki promtail grafana/provisioning/datasources grafana/provisioning/dashboards
```

2. **创建配置文件**

将上面的配置文件分别保存到对应的目录中。

3. **启动服务**

```bash
docker-compose up -d
```

4. **访问服务**

- Prometheus: http://localhost:9090
- Grafana: http://localhost:3000 (用户名/密码: admin/admin)
- Jaeger: http://localhost:16686

## 集成与使用

### 在Grafana中添加数据源

1. 登录Grafana后，点击左侧菜单的"Configuration" -> "Data sources"
2. 点击"Add data source"
3. 分别添加以下数据源：
   - Prometheus: URL填写 `http://prometheus:9090`
   - Loki: URL填写 `http://loki:3100`
   - Jaeger: URL填写 `http://jaeger:16686`

### 创建仪表盘

1. 在Grafana中，点击左侧菜单的"Create" -> "Dashboard"
2. 点击"Add new panel"
3. 选择相应的数据源，编写查询语句，创建可视化图表

### 查看分布式追踪

1. 访问Jaeger UI: http://localhost:16686
2. 在服务列表中选择要查看的服务
3. 点击"Find Traces"查看追踪数据
4. 点击具体的追踪查看详细信息

### 查看日志

1. 在Grafana中，点击左侧菜单的"Explore"
2. 选择Loki数据源
3. 使用LogQL查询语言查询日志

## 最佳实践

1. **合理设置标签**：为Prometheus和Loki设置有意义的标签，便于查询和过滤
2. **配置适当的 retention 策略**：根据存储容量和业务需求，设置合理的数据保留时间
3. **使用告警规则**：为关键指标设置告警规则，及时发现问题
4. **定期备份**：定期备份监控数据，防止数据丢失
5. **优化查询**：避免复杂的查询影响系统性能
6. **监控系统本身**：监控可观察性系统本身的健康状态

## 总结

通过Prometheus、Loki、Jaeger和Grafana的组合，我们可以构建一个完整的可观察性平台，实现对系统的全方位监控。这种组合不仅可以帮助我们快速发现和解决问题，还可以通过数据分析优化系统性能，提高系统的可靠性和可用性。

以上部署方案提供了一个基础的可观察性环境，你可以根据实际需求进行调整和扩展。例如，添加更多的exporter来监控特定的服务，配置更复杂的告警规则，或者集成更多的数据源。
