## Prometheus - 指标监控系统

### 架构概述

Prometheus是一个开源的监控系统和时间序列数据库，主要用于收集和存储系统指标。其核心架构包括：

1. **Prometheus Server**：负责抓取和存储时间序列数据
2. **Exporters**：负责将各种系统和应用的指标转换为Prometheus可识别的格式
3. **Alertmanager**：处理告警并发送通知
4. **Pushgateway**：用于临时任务的指标收集

Prometheus采用拉取（pull）模式获取指标，通过HTTP协议从目标服务中抓取数据。它使用一种名为PromQL的查询语言来分析和查询指标数据。

### 核心组件

- **Prometheus Server**：核心组件，包含数据采集、存储和查询功能
- **Service Discovery**：自动发现需要监控的目标
- **TSDB**：内置的时间序列数据库，用于存储指标数据
- **Alerting Rules**：定义告警规则

## Loki - 日志聚合系统

### 架构概述

Loki是由Grafana Labs开发的一个水平可扩展、高可用性的日志聚合系统。它的设计理念是"像Prometheus一样处理日志"，具有以下特点：

1. **标签索引**：与Prometheus类似，Loki使用标签来索引日志流，而不是全文索引
2. **高效存储**：通过压缩和去重，减少存储成本
3. **与Grafana集成**：无缝集成到Grafana中，提供统一的可视化体验

### 核心组件

- **Promtail**：日志收集代理，负责将日志发送到Loki
- **Loki**：主服务器，负责存储和索引日志
- **Grafana**：用于查询和可视化日志

Loki的架构设计使其特别适合处理大规模的日志数据，同时保持查询性能和存储效率的平衡。

## Jaeger - 分布式追踪系统

### 架构概述

Jaeger是一个开源的分布式追踪系统，用于监控和排查微服务架构中的性能问题。它由Uber开发并开源，后来成为CNCF的毕业项目。

Jaeger的核心架构包括：

1. **Jaeger Client**：集成到应用程序中的客户端库，用于生成和发送追踪数据
2. **Jaeger Collector**：接收和处理追踪数据
3. **Jaeger Query**：提供UI界面，用于查询和可视化追踪数据
4. **Storage**：存储追踪数据，支持多种后端如Cassandra、Elasticsearch等

### 核心组件

- **Agent**：运行在每个主机上，接收本地客户端发送的追踪数据并批量发送到Collector
- **Collector**：接收追踪数据，进行处理和聚合，然后存储到后端存储
- **Query**：提供API和UI，用于查询和展示追踪数据
- **Ingester**：处理和索引追踪数据（在某些存储后端中使用）

## Grafana - 可视化平台

### 架构概述

Grafana是一个开源的监控和可视化平台，支持多种数据源，包括Prometheus、Loki、Jaeger等。它提供了丰富的可视化选项和强大的查询编辑器。

Grafana的核心架构包括：

1. **Grafana Server**：主服务器，提供Web界面和API
2. **Data Sources**：数据源配置，连接到各种监控系统
3. **Dashboards**：可视化仪表盘，展示监控数据
4. **Alerting**：告警系统，基于监控数据触发告警

### 核心功能

- **统一界面**：为不同的监控系统提供统一的可视化界面
- **灵活的仪表盘**：支持多种图表类型和布局
- **强大的查询编辑器**：针对不同数据源的专用查询编辑器
- **告警管理**：集中管理来自不同系统的告警

## 完整的Docker Compose部署方案

下面提供一个完整的Docker Compose配置，用于部署Prometheus、Loki、Jaeger和Grafana的集成环境：
实际上有些配置我并不清楚，以下配置由ai生成，需要根据实际情况进行调整。我自己仅仅使用了一些基本的配置，一般来说，作为学习使用，像loki，jaeger，prometheus的具体配置直接默认即可，第一次使用知道otel的各个端口有什么用，以及如何配置即可。

```yaml
version: '3.8'

services:
  # Prometheus 服务 - 指标监控系统
  prometheus:
    image: prom/prometheus:latest  # 使用最新版本
    container_name: prometheus
    restart: always
    volumes:
      - prometheus_data:/prometheus  # 仅挂载数据卷，使用默认配置
    ports:
      - "9090:9090"  # Prometheus Web UI 端口
    networks:
      - monitoring

  # Grafana 服务 - 可视化平台
  grafana:
    image: grafana/grafana:latest  # 使用最新版本
    container_name: grafana
    restart: always
    volumes:
      - grafana_data:/var/lib/grafana  # 仅挂载数据卷，使用默认配置
    environment:
      - GF_SECURITY_ADMIN_USER=admin  # 管理员用户名
      - GF_SECURITY_ADMIN_PASSWORD=admin  # 管理员密码
    ports:
      - "3000:3000"  # Grafana Web UI 端口
    networks:
      - monitoring

  # Loki 服务 - 日志聚合系统
  loki:
    image: grafana/loki:latest  # 使用最新版本
    container_name: loki
    restart: always
    volumes:
      - loki_data:/loki  # 仅挂载数据卷，使用默认配置
    ports:
      - "3100:3100"  # Loki API 端口
    networks:
      - monitoring

  # Jaeger 服务 - 分布式追踪系统
  jaeger:
    image: jaegertracing/all-in-one:latest  # 使用最新版本
    container_name: jaeger
    restart: always
    ports:
      - "16686:16686"  # Jaeger Web UI 端口
      - "14268:14268"  # Jaeger HTTP 收集器端口
    networks:
      - monitoring

# 数据卷定义
volumes:
  prometheus_data:  # Prometheus 数据存储
  grafana_data:     # Grafana 数据存储
  loki_data:        # Loki 数据存储

# 网络定义
networks:
  monitoring:  # 监控网络，用于服务间通信
    driver: bridge
```

### 配置说明

由于我们使用了各服务的默认配置，因此不需要创建额外的配置文件。以下是各服务的默认配置说明：

1. **Prometheus**：使用默认配置，会自动监控自身
2. **Grafana**：通过环境变量设置了管理员用户名和密码
3. **Loki**：使用默认配置，提供日志存储和查询功能
4. **Jaeger**：使用all-in-one模式，集成了所有组件

如果需要自定义配置，可以在docker-compose.yml中添加相应的volumes挂载，例如：

```yaml
# 示例：自定义Prometheus配置
prometheus:
  # ...
  volumes:
    - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
    - prometheus_data:/prometheus
```

## 集成与使用

### 在Grafana中添加数据源

1. 登录Grafana后，点击左侧菜单的"Configuration" -> "Data sources"
2. 点击"Add data source"
3. 分别添加以下数据源：
   - Prometheus: URL填写 `http://prometheus:9090`  # 容器内部访问地址
   - Loki: URL填写 `http://loki:3100`          # 容器内部访问地址
   - Jaeger: URL填写 `http://jaeger:16686`      # 容器内部访问地址

### 创建仪表盘

1. 在Grafana中，点击左侧菜单的"Create" -> "Dashboard"
2. 点击"Add new panel"
3. 选择相应的数据源，编写查询语句，创建可视化图表

### 查看分布式追踪

1. 访问Jaeger UI: http://localhost:16686
2. 在服务列表中选择要查看的服务
3. 点击"Find Traces"查看追踪数据
4. 点击具体的追踪查看详细信息

### 查看日志

1. 在Grafana中，点击左侧菜单的"Explore"
2. 选择Loki数据源
3. 使用LogQL查询语言查询日志
